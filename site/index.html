01

Everything tuned to one palette

Five ideas, executed all the way down the boot chain.

DESKTOP

Quiet Ferric desktop

Hyprland + waybar + kitty + fuzzel + mako, tuned to oxide #D85A30 on near-black #0C0B0A. 1px seams. No blur. No glow.

BOOT

Continuous boot chain

Syslinux menu, Plymouth tape-spool splash, tuigreet, and the desktop all share the same instrument-panel look. No jarring handoffs from power button to prompt.

INSTALL

Two installers

ferric-install — preseeded archinstall in the terminal. Or Install FerricOS — Calamares, graphical, with ext4 / btrfs / LUKS options.

BASE

Real Arch underneath

pacman, AUR-compatible, rolling release. The distro identity travels in a handful of ferric-* packages from this repo's own pacman repo — nothing forked, nothing frozen.

BUILD

Everything reproducible

One repository is the entire distro: the archiso recipe, every ferric-* package source, and the CI that builds it. Rebuild the ISO and every package yourself, locally or in CI.

05

Get it on metal

Five steps from download to desktop.

  1. 1

    Download & verify

    Grab the latest ISO from Releases and check it against SHA256SUMS.

  2. 2

    Write it to a USB stick

    sudo dd if=ferricos-*.iso of=/dev/sdX bs=4M status=progress oflag=sync

    …or just drop the ISO onto a Ventoy stick.

  3. 3

    Disable Secure Boot

    Not yet supported. Flip it off in firmware settings.

  4. 4

    Boot the stick

    BIOS or UEFI. The live session logs straight into Hyprland as the ferric user.

  5. 5

    Install

    Install FerricOS (app launcher → Calamares), or run ferric-install in a terminal for preseeded archinstall.

Prefer to build it yourself?

On an up-to-date Arch system with archiso, base-devel, qemu-full, and edk2-ovmf:

./scripts/build-packages-local.sh && ./scripts/build-iso.sh

The recipe is evergreen — the ISO is a snapshot. Test in QEMU with ./scripts/test-bios.sh or ./scripts/test-uefi.sh.

Rules of the house

The architecture is opinionated so the desktop can be quiet.

  • The live overlay is live-only. Nothing in the ISO's airootfs ever reaches an installed system.
  • Everything an install receives travels as a package. Rice in ferric-skel, artwork in ferric-artwork, identity in ferric-base.
  • One file, one home. When a file graduates into a package, it moves — it never lives in two places.
  • Never fight Arch. No forked core packages, no held-back kernels, no shipping over filesystem's files.
  • The distro definition excludes hardware. NVIDIA, microcode, and personal apps stay out of ferric-base.
07

Fine print

Is this a fork of Arch?

No. It's Arch, delivered. Installed systems update straight from Arch's official mirrors — kernel, desktop, everything. The FerricOS identity lives in a handful of extra packages.

Do I ever need to reinstall?

No. ISOs are only a delivery vehicle for new installs. Existing systems roll forward forever with pacman -Syu.

How do updates work?

pacman -Syu pulls Arch's official repos plus the [ferric] repo. FerricOS packages are republished by CI whenever their sources change; ISOs are rebuilt monthly.

Can I use the AUR?

Yes — it's real Arch underneath. Any AUR helper works exactly as it would on vanilla Arch.

Secure Boot?

Not yet supported — disable it before booting the stick.

Can I fork it?

Yes — it's GPL-3.0-or-later. But forks must rename: "FerricOS", the wordmark, and the artwork identify this project. Derivative distros should ship their own name and branding.

Who makes this?

FerricOS is designed & built by Cam Garrison — all the code is his. It's an independent personal project, not affiliated with or endorsed by Arch Linux. Built on Arch Linux and archiso; installer flow modeled on EndeavourOS.
